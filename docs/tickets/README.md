# Tickets

Create one markdown file per ticket using IDs like:
- `site-e001.md` for epics
- `site-f001.md` for features
- `site-s001.md` for stories
- `site-t001.md` for tasks

Each ticket should include:
- problem
- value
- scope
- dependencies
- acceptance criteria
- risks
- open questions
- definition of done
