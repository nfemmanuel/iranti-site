# Site Evidence And Claims Protocol

## Purpose
Keep the site technically credible by tying proof and product language to current evidence.

## Rules
1. The installed Iranti product surface is the source of truth for how Iranti works today.
2. Benchmark claims must track the current audited state of each benchmark.
3. Mixed evidence must be presented as mixed evidence.
4. Null results, yellow tracks, and blocked tracks must not be smoothed into green claims.
5. If current evidence is weaker than a draft claim, weaken the claim.

## Required Review For Proof-Oriented Work
1. what changed
2. evidence used
3. valid claims
4. claims not yet supportable
5. known risks or caveats
